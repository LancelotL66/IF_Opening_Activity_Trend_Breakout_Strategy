# IF 开盘活跃趋势突破策略

这是一个基于 DQT Python Toolbox (`dqtrader`) 的股指期货日内趋势突破策略项目，包含两个版本：

- 单合约版本：只交易沪深 300 股指期货主力连续合约 `CFFEX.IF0000`
- 多合约版本：同时交易 `IF`、`IC`、`IH`、`IM` 四个股指期货主力连续合约，并按标的做等权资金分配

策略核心思想是：在每日开盘后的前 6 根 5 分钟 K 线内判断市场是否出现“活跃开盘”。如果开盘成交额相对历史均值明显放大，并且开盘区间振幅达到阈值，则认为当天具备日内趋势突破机会。之后价格向上突破开盘区间高点时做多，向下跌破开盘区间低点时做空，并使用动态仓位、固定止损和尾盘平仓控制风险。

## 策略参数

两个策略的主要参数都集中在脚本顶部，便于直接调整：

```python
INITIAL_CASH = 10_000_000
BEGIN_DATE = "2023-01-01"
END_DATE = "2026-06-07"

OPEN_BARS = 6
LOOKBACK_DAYS = 20
BARS_PER_DAY = 48
AMOUNT_RATIO_THRESHOLD = 1.1
RANGE_THRESHOLD = 0.002
STOP_LOSS_PCT = 0.01
MAX_TRADE_COUNT = 5
```

多股指期货版本额外定义交易标的和基础权重：

```python
TARGET_LIST = [
    "CFFEX.IF0000",
    "CFFEX.IC0000",
    "CFFEX.IH0000",
    "CFFEX.IM0000",
]

BASE_TARGET_WEIGHT = 1.0 / len(TARGET_LIST)
```

| 参数 | 说明 |
| --- | --- |
| `OPEN_BARS` | 开盘区间 K 线数量，5 分钟频率下 6 根约为 30 分钟 |
| `LOOKBACK_DAYS` | 计算历史开盘成交额均值的回看天数 |
| `BARS_PER_DAY` | 每日 5 分钟 K 线数量估计值，用于确定历史数据窗口长度 |
| `AMOUNT_RATIO_THRESHOLD` | 当日开盘成交额相对历史均值的活跃阈值 |
| `RANGE_THRESHOLD` | 开盘区间振幅阈值 |
| `STOP_LOSS_PCT` | 固定止损比例 |
| `MAX_TRADE_COUNT` | 单标的单日最大开仓次数 |
| `BASE_TARGET_WEIGHT` | 多标的版本中每个合约的基础资金权重 |

## 策略逻辑

### 开盘活跃判断

每个交易日开盘后，策略取前 `OPEN_BARS` 根 5 分钟 K 线，计算：

- 开盘区间高点
- 开盘区间低点
- 开盘区间振幅
- 开盘区间成交额

再用过去 `LOOKBACK_DAYS` 个交易日的开盘成交额均值作为基准。当以下条件同时满足时，认定为活跃开盘：

```text
开盘成交额 / 历史开盘成交额均值 >= AMOUNT_RATIO_THRESHOLD
开盘区间振幅 >= RANGE_THRESHOLD
```

### 动态仓位

策略根据开盘成交额放大倍数和开盘振幅设置日内仓位：

| 条件 | 原始仓位 |
| --- | ---: |
| 成交额倍数 >= 1.8 且振幅 >= 0.004 | 100% |
| 成交额倍数 >= 1.3 且振幅 >= 0.003 | 80% |
| 其他活跃开盘 | 50% |

单合约版本直接使用上述仓位。多股指期货版本会再乘以 `BASE_TARGET_WEIGHT`，即四个标的默认各占 25% 基础资金权重，避免多个合约同时触发时总仓位失控。

### 入场与出场

入场规则：

- 当前价格突破开盘区间高点：做多
- 当前价格跌破开盘区间低点：做空

风控规则：

- 多头价格跌破 `开盘高点 * (1 - STOP_LOSS_PCT)` 止损
- 空头价格突破 `开盘低点 * (1 + STOP_LOSS_PCT)` 止损
- `14:55` 之后尾盘平仓
- 单标的单日最多开仓 `MAX_TRADE_COUNT` 次

## 多股指期货版本说明

多股指期货版本不是简单把 IF 信号复制到所有合约，而是对每个合约分别维护日内状态：

- 每个合约独立计算开盘成交额倍数和开盘振幅
- 每个合约独立判断是否属于活跃开盘
- 每个合约独立记录开盘区间高低点、持仓方向和当日交易次数
- 每个合约独立执行突破、止损和尾盘平仓
- 仓位在单合约动态仓位基础上乘以等权基础权重

这种设计可以降低单一股指期货品种的路径依赖，并让 IF、IC、IH、IM 在不同市场风格下分别贡献信号。

## 注意事项

- 本策略依赖 DQT 客户端权限，离线状态下无法正常获取行情或运行回测。
- 交易标的是股指期货主力连续合约，实盘前需要确认合约映射、保证金、手续费和交易权限。
- 当前使用 5 分钟 K 线，`BARS_PER_DAY = 48` 是按日内可用 K 线数量估算。
- 策略为日内策略，尾盘会尝试平仓，不设计隔夜持仓。
- 代码未额外处理成交失败、盘口流动性不足、极端滑点、涨跌停无法成交等实盘细节。
- 代码文件包含中文，建议使用 UTF-8 编码打开和提交。
